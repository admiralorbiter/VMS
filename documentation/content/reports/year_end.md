# District Year-End Report

District-scoped annual summaries for grant reporting and stakeholder communications.

---

## Overview

**Location:** `Reports > District Year-End`
**URL:** `/reports/district/year-end`
**Who uses it:** PrepKC staff, district coordinators
**Primary use:** End-of-year grant reporting, district feedback packets, stakeholder presentations

The Year-End Report aggregates all completed events for a given academic year across each
partner school district. Data is cache-backed for fast load times — each district's numbers
are pre-computed and stored, then refreshed on demand.

---

## How to Use

### 1. Select a School Year

Use the **School Year** dropdown at the top of the page to select the academic year
(e.g., `2024-25`). The page defaults to the current school year.

### 2. Browse District Cards

Each card on the main page shows a district's top-line numbers for the selected year:
- Total events
- Total students reached
- Total volunteer hours
- Total unique volunteers

Click a district card to open its **Detail Page**.

### 3. Drill Into a District

The detail page shows the full breakdown for a single district:

| Section | What it shows |
|---------|---------------|
| **Summary Metrics** | Events, students, volunteers, hours, organizations, schools served |
| **Event Type Filter** | Filter the data table by clicking interactive event type pill buttons |
| **Activity Breakdown** | A consolidated table of all events, toggleable between 'All Events' and 'By School' |

### 4. Export to Excel

Click **Download Excel** on any district's detail page to export a `.xlsx` file with four sheets:

| Sheet | Contents |
|-------|----------|
| **Summary** | Top-line metrics for the year |
| **Event Types** | Count by event type category |
| **Monthly Breakdown** | Month-by-month aggregates |
| **Events Detail** | One row per event with date, type, students, volunteers, hours |

> [!TIP]
> Run the export **after** using "Refresh Data" to ensure the numbers are current before
> distributing to stakeholders.

### 5. Refresh Data

Click the **Refresh Data** button to force a full recompute of all district stats for the
selected year. Use this:
- After a Salesforce data sync that affects the year being reported
- If numbers look stale or incomplete

Refresh can take 30–60 seconds for a full year. The page shows the last-updated timestamp
so you can confirm data freshness.

## Metric Definitions

The Year-End report tracks data across three distinct tiers. While general event metrics are computed from standard event attendance, specialized programs and district-specific credentials require manual entry.

### 1. General Auto-Computed Metrics
These core metrics are calculated automatically from completed events and participation records.

| Metric | Definition | Notes |
|--------|------------|-------|
| **Total Events** | Count of all completed events | Excludes cancelled and draft events |
| **Total Students Reached** | Sum of student attendance counts | Virtual sessions use estimated counts |
| **Unique Students** | Count of distinct student IDs | Virtual sessions excluded from unique count |
| **Total Volunteer Engagements** | Sum of attended volunteer participations | One volunteer attending 3 events = 3 engagements |
| **Unique Volunteers** | Count of distinct volunteer IDs | |
| **Total Volunteer Hours** | Sum of `delivery_hours` | Excludes no-shows and cancellations |

### 2. Auto-Computed Program Breakdown
These program-specific totals are automatically generated by filtering events by their `EventType`:
- ✅ Career Jumping
- ✅ Career Speakers
- ✅ Career Fair / College Fair (High School only)
- ✅ HealthStart
- ✅ BFI Offsite Visits
- ✅ Connector Sessions (Teachers Engaged, Students Participated, Total Sessions)
- ✅ DIA Students (Middle School)
- ✅ SLA Students
- ✅ Client Connected Projects
- ✅ P2T Students
- ✅ P2GD Students
- ✅ Math Relays (Total Event Count)

### 3. Manual Input Metrics
Because certain programs (like credential outcomes and district-specific awards) do not map 1:1 with event attendance, they must be manually entered before running the final report.

Staff must navigate to the **Input Data** view (accessible via the `Detailed Stats Breakdown` page) to populate these fields:
- ✍️ **Math Relays Splits:** HS vs JR participant counts, and HS/MS awards counts
- ✍️ **Credentials & Credits:** DIA IRC Earned, P2GD Dual Credit, P2T Completions, CNA/CMA Students, College Credits (Students & Total)
- ✍️ **Special Programs:** Careers on Wheels (Center 5th Graders) and Holland 1916 Visits (KCKPS)

> [!NOTE]
> **Connector Sessions** are included in the enhanced stats breakdown but excluded from
> the main event list and monthly timeline. This is intentional — they are tabulated
> separately in the Event Type Breakdown section.

## Filters

| Filter | Options | Effect |
|--------|---------|--------|
| **School Year** | Dropdown — `2020-21` through current+1 | Scopes all data to the selected academic year (Aug 1 – Jul 31) |
| **Host Filter** | `All` / `PrepKC Only` | `PrepKC Only` restricts to events where PrepKC is the session host |

---

## Known Limitations

- **Virtual student uniqueness:** Individual student tracking is not possible for virtual
  sessions (students are estimated from teacher class sizes). Unique student counts
  reflect in-person attendance only.
- **District attribution:** Events are attributed to a district based on the event's
  `district_partner` field, linked schools, and configured district aliases. Edge cases
  where an event is uncategorized may not appear.
- **Cache lag:** Numbers reflect the state of data at the last cache refresh, not
  real-time Salesforce data. Always refresh before distributing final reports.
