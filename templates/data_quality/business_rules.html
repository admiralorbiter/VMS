Volunteer Entity

Contact records with volunteer activities and skills

Required Fields Validation
Core fields that must be populated for all volunteer records
  • first_name - First name of the volunteer (required)
  • last_name - Last name of the volunteer (required)
  • email - Email address for communication (required)
Implemented Business rule validation with error severity
Count Validation
Context-aware validation comparing VMS volunteer count with Salesforce

Expected Threshold: 5.0% discrepancy tolerance

Context Analysis: Considers seasonal variations, recent events, and data import timing

Scoring: Graduated scoring based on business context

Implemented Full context-aware validation
Field Completeness
Validation of required field population rates

Thresholds:

  • Info: ≥90% population
  • Warning: ≥75% population
  • Error: ≥50% population
  • Critical: <50% population
Implemented Configurable severity thresholds
Data Type Validation
Ensuring data types match expected formats

Validation:

  • Email format validation
  • String length validation
  • Date format validation
Implemented Type checking with severity-based scoring
Relationship Validation
Validation of optional relationship field population

Optional Relationships:

  • AccountId - Organization association
  • npsp__Primary_Affiliation__c - Primary affiliation

Validation: Reports population rates for optional fields

Implemented Population rate analysis
